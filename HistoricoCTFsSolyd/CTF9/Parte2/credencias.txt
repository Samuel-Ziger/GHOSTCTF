Nota importante
---------------

No `.env` do Langflow costuma aparecer algo como
`LANGFLOW_SUPERUSER_PASSWORD=blogo123#` (com `#` no fim). Na prática, o hash
guardado na base SQLite (bcrypt) correspondia à password **blogo123** sem o
carácter `#`. Por isso, se `blogo123#` falhar no login, tenta **só**
`blogo123`.


Login via API (JWT)
-------------------

Substitui ALVO_IP pelo IP do lab. Mantém o cabeçalho Host a apontar para o
Langflow por trás do nginx.

  curl -sS -X POST 'http://ALVO_IP/api/v1/login' \
    -H 'Host: flow.projects-blogo.sy' \
    -H 'Content-Type: application/x-www-form-urlencoded' \
    -H 'Accept: application/json' \
    --data-binary 'username=admin&password=blogo123&grant_type=password'

A resposta traz `access_token` e `refresh_token` (Bearer nas rotas `/api/v1/...`).
